# Open-Sesame v0.3.0-native-lite

Pre-release native Android build for authorized EuroPark (autoparkki) QR door workflows in Finland.

## Highlights

- Native Kotlin Android implementation is now the active app line.
- Root repository layout now opens directly as the Android Gradle project.
- Main screen uses a clearer step-card layout: `SCAN`, `ENTER`, then `OPEN`.
- `SCAN`, `ENTER`, and `OPEN` are visually emphasized as the primary workflow.
- Secondary controls are de-emphasized: `EDIT`, `DELETE`, `DEBUG`, `UPDATE`, and `CLEAR`.
- Default local plate is now `ABC-123` when no plate exists.
- Right-top `?` help popup explains the EuroPark/autoparkki quick-opener behavior.
- `DEBUG` popup now includes `Mode: real opener` and current-door webpage/parser diagnostics.
- `UPDATE` opens the GitHub releases page.
- User-facing operator wording is standardized as `EuroPark (autoparkki)`.

## Scope and limitations

- Use only with door URLs and license plates that you are authorized to use.
- The app does not bypass EuroPark access control.
- The app sends the same web request expected by the legacy autoparkki page.
- The app does not physically verify whether the door opened.
- A success message means the HTTP/form flow looked successful or was sent; the user must still verify the physical door.
- This build is currently intended for EuroPark/autoparkki access pages in Finland.
